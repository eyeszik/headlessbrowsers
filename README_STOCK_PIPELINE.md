# Stock Image Preparation Pipeline

**AI-powered stock photography workflow: generate → upscale → optimize → organize**

Automated pipeline that generates AI images, upscales them to stock-platform requirements (6000×4000, 300 DPI), generates SEO-optimized metadata, and organizes everything in Notion for manual upload to Shutterstock, Adobe Stock, Getty Images, and iStock.

## Overview

This pipeline automates the tedious work of preparing AI-generated images for stock photography platforms:

```
┌─────────────────┐
│ Generate Images │  DALL-E 3 / Stable Diffusion / Mock
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│  Deduplicate    │  SHA-256 hash deduplication
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│    Upscale      │  6000×4000, 300 DPI, sRGB JPEG
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│ Generate Meta   │  AI SEO optimization (titles/descriptions/keywords)
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│  Save Locally   │  JPEG + .txt metadata files
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│ Upload to Notion│  Organized database (optional)
└─────────────────┘
         │
         ▼
    Manual Upload   ← You manually upload to stock platforms
```

**Key principle:** Human-in-the-loop compliance — no automated uploads to stock platforms (ToS compliant).

## Features

### ✅ Multi-Provider Image Generation
- **OpenAI DALL-E 3** — highest quality photorealistic images
- **Stability AI SDXL** — fast, cost-effective, good for artistic styles
- **Mock provider** — testing without API keys

### ✅ Stock-Platform-Ready Upscaling
- **Resolution:** 6000×4000 pixels (24 MP) — exceeds all platform minimums
- **DPI:** 300 DPI embedded metadata
- **Color:** sRGB ICC profile
- **Format:** High-quality JPEG (quality=95)
- **Backend:** Real-ESRGAN (GPU) or Pillow (CPU fallback)

### ✅ AI-Powered SEO Metadata
- **Titles:** 50–70 chars, keyword-rich, human-readable
- **Descriptions:** 200–500 chars, narrative with commercial use cases
- **Keywords:** 30–50 tags, hierarchical (broad → specific → conceptual)
- **AI Providers:** OpenAI GPT-4, Anthropic Claude, Google Gemini
- **Platform Validation:** Auto-checks Shutterstock, Adobe Stock, Getty, iStock constraints

### ✅ Notion Workspace Organization
- Rich database with 13 properties (category, style, keywords, color palette, etc.)
- Embedded metadata text blocks
- Upload status tracking
- Target platform multi-select
- Confidence scoring

### ✅ Production-Ready Infrastructure
- Async/await throughout for performance
- Rate limiting (Notion 3 req/s compliance)
- SHA-256 integrity verification
- Exponential backoff retry logic
- Graceful degradation (works without API keys)
- Comprehensive error handling

- Ready for manual upload to stock platforms

## Architecture

### Pipeline Stages

| Stage | Purpose | Can Fail? | Rollback? |
|-------|---------|-----------|-----------|
| **1. Generate** | Create AI images | Yes | N/A |
| **2. Deduplicate** | Remove SHA-256 duplicates | No | N/A |
| **3. Upscale** | Resize to 6000×4000 | Yes | Skip image |
| **4. Metadata** | Generate SEO data | No* | Uses heuristic fallback |
| **5. Save** | Write JPEG + .txt | Yes | Skip image |
| **6. Notion** | Upload to workspace | Yes | Skip (images still saved locally) |

*Metadata generation always succeeds — falls back to heuristic templates if all AI providers fail.

### File Structure

```
data/
├── stock_images/
│   └── <batch-id>/
│       ├── <img-001>_upscaled.jpg  (6000×4000, 300 DPI)
│       ├── <img-002>_upscaled.jpg
│       └── ...
└── metadata/
    └── <batch-id>/
        ├── <img-001>_metadata.txt
        ├── <img-002>_metadata.txt
        └── ...

stock_pipeline_report_<run-id>.json  ← Full results
```

## Troubleshooting

### Issue: "ModuleNotFoundError: No module named 'google.generativeai'"

**Solution:**
```bash
pip install google-generativeai==0.3.2
```

### Issue: "notion-client is required"

**Solution:**
```bash
pip install notion-client==2.2.1
```

### Issue: Images are blurry after upscaling

**Cause:** Pillow (CPU) upscaler has lower quality than Real-ESRGAN (GPU).

**Solutions:**
1. Use a GPU and install Real-ESRGAN:
   ```bash
   pip install realesrgan basicsr
   UPSCALER_BACKEND=realesrgan
   ```
2. Generate larger images initially:
   ```bash
   IMAGE_GEN_SIZE=1792x1024  # DALL-E 3 HD mode
   ```

### Issue: "Notion API returned 401 Unauthorized"

**Causes:**
1. Invalid `NOTION_API_TOKEN`
2. Integration not shared with target page

**Solution:**
```bash
# 1. Verify token in .env
echo $NOTION_API_TOKEN

# 2. Share page with integration:
#    Page → ⋯ → Connections → Add your integration
```

### Issue: Metadata is generic/low-quality

**Causes:**
1. Using heuristic fallback (no API keys)
2. Low-quality prompt

**Solutions:**
```bash
# 1. Add AI provider key
GEMINI_API_KEY=your-key

# 2. Use detailed prompts
python -m backend.app.workflows.stock_prep_pipeline \
  --theme "female entrepreneur" \
  --context "working on laptop in modern coffee shop, natural window light, candid moment, shallow depth of field" \
  --count 5
```

### Issue: Pipeline hangs during generation

**Causes:**
1. Network timeout
2. API rate limiting

**Solutions:**
```bash
# Check API status
curl http://localhost:8000/api/v1/stock-images/status

# Run with fewer images
--count 5  # Instead of 50

# Check logs
tail -f backend.log
```

## FAQ

### Q: Can I use this to automatically upload to stock platforms?

**A:** No. Automated uploads violate stock platform Terms of Service. This pipeline prepares everything (images + metadata), but you manually upload via the platform's web interface. This is intentional and compliant.

### Q: Do I need to disclose that images are AI-generated?

**A:** Yes. Most platforms require disclosure:
- **Shutterstock:** Tag as "AI-generated"
- **Adobe Stock:** Select "Generative AI" during upload
- **Getty:** Currently not accepting AI images (as of Jan 2025)
- **iStock:** Follow parent company (Getty) policy

The pipeline automatically marks images as AI-generated in metadata.

### Q: What image resolution do stock platforms require?

| Platform | Minimum | Recommended | This Pipeline |
|----------|---------|-------------|---------------|
| **Shutterstock** | 4 MP (2000×2000) | 6 MP+ | 24 MP (6000×4000) ✓ |
| **Adobe Stock** | 4 MP | 6 MP+ | 24 MP ✓ |
| **Getty** | 6 MP | 10 MP+ | 24 MP ✓ |
| **iStock** | 3.15 MP | 6 MP+ | 24 MP ✓ |

**Answer:** This pipeline exceeds all requirements at 6000×4000 (24 MP).

### Q: How long does it take to generate 10 images?

**Breakdown:**
- **Generation:** ~30s (DALL-E 3) or ~5s (Stability AI)
- **Upscaling:** ~5s/image (CPU) or ~2s/image (GPU)
- **Metadata:** ~2s/image (Gemini) or ~5s/image (GPT-4)
- **Notion upload:** ~3s/image

**Total for 10 images:**
- **DALL-E 3 + Gemini + CPU:** ~7 minutes
- **Stability + Gemini + GPU:** ~1.5 minutes

### Q: Can I run this without Notion?

**A:** Yes. Set:
```bash
NOTION_API_TOKEN=  # Leave empty
```

Images and metadata files are still saved locally. Skip Notion upload.

### Q: What's the best provider combination?

**For maximum profit:**
- **Images:** Stability AI ($0.002/image)
- **Metadata:** Gemini ($0.0001/image)
- **Total:** $0.002/image → need 1 download to profit

**For maximum quality:**
- **Images:** DALL-E 3 ($0.04/image)
- **Metadata:** GPT-4 ($0.003/image)
- **Total:** $0.043/image → need 2–3 downloads to profit

**For testing:**
- **Images:** Mock ($0)
